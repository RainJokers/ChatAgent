# 🤖 智能聊天助手 Agent

一个基于 **管道-过滤器 + 分层架构** 混合风格的智能对话系统，支持本地规则策略与大模型（LLM）双模式。

## ✨ 功能特性

| 功能 | 说明 |
|------|------|
| 💬 **自由对话** | 支持自然语言聊天，Agent 智能回复 |
| 🔍 **意图识别** | 自动识别 4 类意图：闲聊 / 知识查询 / 任务创建 / 情绪倾诉 |
| 😊 **情绪感知** | 检测积极/中性/消极情绪，调整回复语气 |
| 📖 **知识检索** | 内置知识库，回答事实性问题 |
| 📋 **任务记录** | 解析"提醒我/记下"类指令，保存待办任务 |
| 💾 **多轮记忆** | 记住对话上下文，支持最多 20 轮记忆 |
| ⚡ **快捷按钮** | 一键发送预设指令 |
| 🔬 **诊断面板** | 显示意图识别、情绪检测、处理耗时、回复模式等信息 |
| 🧠 **大模型接入** | 支持 DeepSeek / OpenAI 等兼容 API，知识库未命中时自动调用 |
| 🔀 **混合策略** | 知识库命中→本地秒回；未命中→大模型智能回答，兼顾速度与质量 |

## 🏗️ 架构设计

### 混合架构风格

```
┌──────────────────────────────────────────────────────┐
│                    交互层 (UI)                        │
│               Streamlit Web 界面                      │
├──────────────────────────────────────────────────────┤
│                    决策层                              │
│  ┌──────────┐  ┌──────────┐  ┌──────────────────┐   │
│  │ 意图识别  │→ │ 情绪检测  │→ │   回复生成        │   │
│  │ (过滤器1) │  │ (过滤器2) │  │ ┌──────────────┐ │   │
│  └──────────┘  └──────────┘  │ │ LLM 策略  │ │   │
│                              │ │ 或规则策略  │ │   │
│                              │ └──────────────┘ │   │
│                            管道-过滤器风格          │
├──────────────────────────────────────────────────────┤
│                    数据层                              │
│   知识库   │   对话历史   │   任务管理   │   JSON    │
│   config.json (LLM 配置)                              │
└──────────────────────────────────────────────────────┘
```

### 设计模式

- **策略模式**（`response_strategies.py`）：5 种策略类（含 LLM 策略），新增意图只需新增策略类
- **观察者模式**（`task_manager.py`）：任务变化时通知 UI 观察者，实现界面更新

## 📁 项目结构

## 🚀 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 运行应用（规则模式）

```bash
streamlit run app.py
```

终端会显示本地地址，默认是 `http://localhost:8501`

### 3. 启用大模型（可选）

编辑 `config.json`，填入 API Key：

```json
{
  "llm": {
    "api_key": "sk-your-key-here",
    "api_base": "https://api.deepseek.com",
    "model": "deepseek-chat"
  }
}
```

支持任意 OpenAI 兼容接口：

| 服务商 | `api_base` | `model` |
|-------|-----------|---------|
| DeepSeek | `https://api.deepseek.com` | `deepseek-chat` |
| OpenAI | `https://api.openai.com/v1` | `gpt-4o` / `gpt-3.5-turbo` |

## 🎮 使用指南

### 输入方式
- **文本输入**：在底部输入框输入文字，按 Enter 发送
- **快捷按钮**：点击左侧面板的快捷指令一键发送

### 示例对话

| 你说 | 规则模式 | 大模型模式 |
|------|---------|-----------|
| "你好！" | 友好问候 | 更自然的对话 |
| "告诉我什么是设计模式" | 从知识库检索（秒回） | 知识库命中→秒回；未命中→LLM 回答 |
| "提醒我明天下午3点开会" | 创建待办任务 | 创建待办 + 自然回复 |
| "今天心情很好！" | 感知积极情绪 | 感知情绪 + 深入交流 |
| "最近压力好大" | 感知消极情绪 | 感知情绪 + 贴心安慰 |

## 🔧 扩展指南

### 新增意图类型

1. 在 `data/intent_rules.json` 中添加关键词规则
2. 在 `response_strategies.py` 中新建策略类（继承 `ResponseStrategy`）
3. 在 `STRATEGY_MAP` 中注册新策略
4. 在 `intent_classifier.py` 的 `classify()` 方法中添加检查顺序

### 扩充知识库

编辑 `data/knowledge_base.json` 的 `facts` 字段即可。知识库中的内容会优先于大模型返回（零成本、秒级响应）。

## 📊 非功能性需求

| 指标 | 目标 | 实现方式 |
|------|------|---------|
| 响应时间（规则模式） | < 0.01s | 纯规则匹配，无外部 API 调用 |
| 响应时间（知识库命中） | < 0.01s | 本地 JSON 检索，不走网络 |
| 响应时间（LLM 模式） | 1-5s | 取决于 API 响应速度 |
| 可扩展性 | 新增意图只需扩展策略类 | 策略模式 + 注册机制 |
| 数据持久化 | JSON 文件存储 | 自动读写 `data/` 目录 |
| LLM 配置 | 编辑 `config.json` | 支持 DeepSeek / OpenAI 兼容接口 |
